# Project agent memory

This file is the project's committed home for project-intrinsic agent knowledge: build, test, release, architecture, and sharp-edge notes that should travel with the code.

- Scenario content (quiz questions + rink animations) lives in `src/data/scenarios/*.ts`; the authoring contract, including puck placement rules, is `src/data/scenarios/AUTHORING.md`. Mechanical invariants are enforced by `npm run validate` (also part of `npm run build`). `freezeLine` goes straight into the situation/question - no "Stop it here"/"Freeze"/"Hold it" transition opener (the whistle + freeze frame already stop the tape).
- The rink renderer draws a carried puck on the carrier's stick: `puckOnStick` in `src/components/RinkDiagram.tsx` shifts any puck within 3.6 units of a player to that player's rim. Author carried pucks at the carrier's exact coordinates and let the renderer handle the offset.
- Voice narration is one coach voice (ElevenLabs), everything spoken, no browser SpeechSynthesis. Each scenario renders three kinds of clip via `npm run narrate` (`scripts/generate-narration.ts`): the animation voice-over (`animation.narration`, manifest key `<id>`), the freeze-line prompt (key `<id>.freeze`), and each read option (key `<id>.opt.<i>`, text built by `optionSpeech` in `scripts/narration-manifest.ts`, matching how SessionScreen used to read them). Files are `public/audio/<key>.<hash>.mp3`; the MP3s + `public/audio/manifest.json` are committed build artifacts, so the app/CI never need a key (`ELEVENLABS_API_KEY` in `.env` only for regeneration; see `.env.example`). Runs are skip-existing/content-hashed and prune-safe without a key. Sharp edge: `contentHash` (in `narration-manifest.ts`) joins fields with a NUL byte (`\x00`), not a space, and folds in the ElevenLabs `voiceSettings` (stability/style/etc.) alongside voice/model/format/text - so retuning the delivery correctly re-renders every clip, but changing the separator or the settings-serialization forces a full (billable) re-render. The client (`src/lib/narrationAudio.ts`) only loads static files - never the API: `AnimatedRink` plays `<id>` during the animation; `SessionScreen` awaits `<id>.freeze` then each `<id>.opt.<i>` during the reveal, and `stop()` cuts audio on sound-off/skip/replay/quit/unmount (the frozen rink's Replay button restarts the animation + narration from the top). Missing audio = silent.
- `npm run lint` is `tsc --noEmit` (only type-checks `src`, not `scripts/`). `npm test` runs the `scripts/*.test.ts` unit tests via `node --test`; the narration generator's logic is tested with the API mocked. Visual/audio changes to animations still need eyeball/ear verification in the running app (`npm run dev`).
