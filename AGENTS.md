# Project agent memory

This file is the project's committed home for project-intrinsic agent knowledge: build, test, release, architecture, and sharp-edge notes that should travel with the code.

- Scenario content (quiz questions + rink animations) lives in `src/data/scenarios/*.ts`; the authoring contract, including puck placement rules, is `src/data/scenarios/AUTHORING.md`. Mechanical invariants are enforced by `npm run validate` (also part of `npm run build`). `freezeLine` goes straight into the situation/question - no "Stop it here"/"Freeze"/"Hold it" transition opener (the whistle + freeze frame already stop the tape).
- The rink renderer draws a carried puck on the carrier's stick: `puckOnStick` in `src/components/RinkDiagram.tsx` shifts any puck within 3.6 units of a player to that player's rim. Author carried pucks at the carrier's exact coordinates and let the renderer handle the offset.
- Voice narration is one coach voice (ElevenLabs), everything spoken, no browser SpeechSynthesis (never reintroduce it). Every scenario is animated and fully voiced, so every mode - including Boss, which draws unfiltered from the whole bank via `pickScenarios` - has audio. `npm run narrate` (`scripts/generate-narration.ts`) renders per-scenario clips (`scenarioClips` in `scripts/narration-manifest.ts`): the animation voice-over (key `<id>`), the freeze-line prompt (`<id>.freeze`), each read option (`<id>.opt.<i>`, text via `optionSpeech`), and the results-beat feedback (`<id>.fb` = correct answer stated + its rationale, via `feedbackSpeech`; for tap the label is spoken verbatim). Plus six scenario-independent openers (`STATIC_OPENER_CLIPS`, keys `fb.correct.<i>`/`fb.wrong.<i>`) threaded into generation/`expectedManifest` via the `extraClips` param. Files are `public/audio/<key>.<hash>.mp3`; the MP3s + `public/audio/manifest.json` are committed build artifacts, so the app/CI never need a key (`ELEVENLABS_API_KEY` in `.env` only for regeneration; see `.env.example`). Runs are skip-existing/content-hashed and prune-safe without a key. Sharp edge: `contentHash` joins fields with a NUL byte (`\x00`), not a space, and folds in the ElevenLabs `voiceSettings` (stability/style/etc.) alongside voice/model/format/text - retuning the delivery correctly re-renders every clip, but changing the separator or the settings-serialization forces a full (billable) re-render. The client (`src/lib/narrationAudio.ts`) only loads static files - never the API: `AnimatedRink` plays `<id>` during the animation; `SessionScreen` awaits `<id>.freeze` then each `<id>.opt.<i>` during the reveal; `FeedbackScreen` plays a rotating opener (`nextFeedbackOpener`) then `<id>.fb` on the results beat for both right and wrong answers; `stop()` cuts audio on sound-off/skip/replay/quit/unmount. Playback timing constants + the pure `fadeRamp` helper live in `src/lib/narrationTiming.ts` (unit-tested by `scripts/narration-timing.test.ts`): `stop()` fades a still-playing clip to silence before pausing (a hard pause mid-waveform clicks/garbles at section boundaries), and `SessionScreen`'s reveal inserts a `OPTION_LEAD_IN_MS` beat after the freeze prompt before reading option A. Missing audio = silent.
- PWA: `vite-plugin-pwa` (config in `vite.config.ts`) makes the app installable and offline-capable. `npm run build` emits the Workbox `generateSW` service worker (`dist/sw.js`) + `dist/manifest.webmanifest`; the plugin auto-injects the manifest link and `registerSW.js` into `index.html`. The app shell (js/css/html/icons/fonts) is precached; `/audio/*` and Google Fonts are runtime-cached (CacheFirst/SWR). Icons in `public/` (`favicon.svg`, `pwa-192x192.png`, `pwa-512x512.png`, `maskable-icon-512x512.png`, `apple-touch-icon.png`) are committed artifacts rasterized from `favicon.svg` by `scripts/generate-icons.mjs` - `sharp` is intentionally not a committed dep, so regenerate with the one-liner in that script's header. To verify installability/offline you must use a real prod build (`npm run build && npm run preview`), not `npm run dev` (SW is disabled in dev by default).
- `npm run lint` is `tsc --noEmit` (only type-checks `src`, not `scripts/`). `npm test` runs the `scripts/*.test.ts` unit tests via `node --test`; the narration generator's logic is tested with the API mocked. Visual/audio changes to animations still need eyeball/ear verification in the running app (`npm run dev`).
